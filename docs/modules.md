# Модули

Три детектора и обратимая анонимизация. Все детекторы работают в памяти, конфиг
(правила/словари) редактируется в админке и применяется без рестарта.

## PII — персональные данные

Двухступенчато: regex находит кандидата, затем алгоритмическая валидация
подтверждает его контрольной суммой — Luhn для банковских карт, mod-11 для ИНН,
mod-101 для СНИЛС, структурная проверка для паспорта РФ (email / телефон / URL /
IP — только regex). Из коробки: email, телефон, URL, IP, банковская карта, СНИЛС,
ИНН, паспорт РФ. Правила (regex по каждому типу) редактируются в админке.

**Ответ:**

```json
{
  "PII_DETECT": true,
  "data": [{"class": "phone", "value": "+79161234567", "start": 12, "end": 24}]
}
```

Сильная сторона — высокая точность (почти нет ложных) на структурированных
данных; имена/адреса не детектим (это зона ML).

## NSFW — обсценная лексика

Поиск по словарю запрещённых слов алгоритмом Aho-Corasick (все слова за один
проход по тексту) с нормализацией (регистр, простые замены символов). Встроенный
словарь — русский мат (`.txt`) плюс английская обсценная лексика (`better_profanity`).
Словари редактируются в админке, можно вести несколько и включать/выключать.

```json
{ "NSFW_DETECT": true }
```

Высокая precision (сработал — значит действительно мат), но recall ограничен:
оскорбления/угрозы без мата словарь не берёт.

## Relevant — релевантность / смолток

Ловит смолток и оффтоп (приветствия, благодарности, прощания и т.п.) по
категориям фраз: срабатывает, когда фразы из категорий покрывают ≥ 80% текста.
Отдельно поддержаны жёсткий флаг для категории `injection` (срабатывает
независимо от покрытия — под prompt-injection; в дефолтный словарь не входит,
заводится в админке) и фильтр «мусорного» ввода (gibberish). Категории (тип +
список фраз) редактируются в админке.

```json
{ "RELEVANT": false, "category": "greeting" }
```

`RELEVANT: false` — пойман чит-чат / инъекция / нерелевантное; `category`
показывает, что именно.

## Анонимизация PII

Замена найденных PII на теги (`<EMAIL_1>`, `<PHONE_1>` …). По умолчанию —
**необратимая** (Redis не используется, восстановление недоступно). Для
обратимости передайте `deanonymize: true` — тогда соответствие «тег → оригинал»
сохраняется в Redis, а в ответе возвращается `id` для последующего
`/v1/deanonymize`.

```json
// POST /v1/anonymize  {"text": "почта ivan@example.com", "deanonymize": true}
{ "id": "6f1c…", "text": "почта <EMAIL_1>" }
```

!!! warning "Redis нужен только для deanonymize"
    При `deanonymize: false` (по умолчанию) Redis не задействован вообще. При
    `deanonymize: true` mapping хранится в Redis (с TTL) — если Redis недоступен,
    сервис вернёт `503`, а не потеряет обратимость молча.
