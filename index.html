    lite-guardrails¶

    lite-guardrails — легковесный self-hosted Guardrails для текстовых данных. Быстрый детерминированный слой на регулярках и словарях (не ML-модель): разворачивается в своём контуре, конфигурируется через админку, отдаёт метрики и пробы.

    Состоит из трёх модулей:

    • PII — находит персональные данные (email, телефон, банковская карта, ИНН, СНИЛС, паспорт РФ, IP, URL) и умеет обратимо их анонимизировать и деанонимизировать.
    • NSFW — ловит мат и обсценную лексику (RU + EN) по словарю.
    • Релевантность — отсекает смолток и оффтоп (приветствия, благодарности, прощания и т.п.).

    Коротко¶

    • Скорость — детекция за доли миллисекунды на CPU (на 2–5 порядков быстрее ML-решений; см. Сравнение).
    • Self-hosted — свой Docker-контур, никаких внешних вызовов.
    • Прозрачно — правила и словари редактируются в админке, изменения применяются без рестарта.
    • Готово к эксплуатации — API-ключи с rate limit, метрики Prometheus, пробы /live /ready, structured-логи без утечки PII, версионированный API /v1.

    • Python SDK — интеграция в несколько строк

    Позиционирование

    lite-guardrails — не «умнее» ML-моделей, а быстрее и предсказуемее. У него почти нет ложных срабатываний на структурированных данных (карты, ИНН, телефоны), но имена/адреса и семантическую токсичность он не ловит — для этого нужен ML-слой. Подробности с цифрами — в разделе Сравнение.
