# Безопасность

## Сообщить об уязвимости

Нашли проблему безопасности — **не открывайте публичный issue**. Напишите на
**m.maksimov@redmadrobot.com** с описанием: как воспроизвести, версия/коммит,
возможные последствия. Постараемся ответить в течение нескольких рабочих дней.

## Что важно при развёртывании

- **Смените дефолты.** `ADMIN_TOKEN` по умолчанию слабый — задайте сильный
  секрет в `.env`. То же для паролей БД. Не запускайте прод с дефолтными
  значениями.
- **Не коммитьте секреты.** `.env` в `.gitignore`; в репозиторий идёт только
  `.env.example` с плейсхолдерами.
- **Ограничьте доступ.** Админку (`/admin/*`) и `/metrics` держите за
  ingress/VPN, наружу не публикуйте. Postgres/Redis — только во внутренней сети.
- **PII в логах.** По умолчанию в аудит-логи пишется анонимизированный текст;
  сырой ввод — только при явном `LOG_RAW_INPUT=true` (включать осознанно).

## Поддерживаемые версии

Патчи безопасности выходят для актуальной ветки `main`. Старые релизы не
сопровождаются.
